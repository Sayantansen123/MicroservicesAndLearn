## **2. MongoDB Query Modifiers**

- **`skip()`** → Skips a specified number of documents before returning results.  
  _Example:_
  ```js
  db.users.find().skip(5);
  ```
- **`sort()`** → Sorts documents in ascending or descending order.  
  _Example:_
  ```js
  db.users.find().sort({ age: -1 }); // -1 = Descending, 1 = Ascending
  ```
- **`count()`** → Counts the number of matching documents.  
  _Example:_
  ```js
  db.users.find().count();
  ```
- **`limit()`** → Limits the number of documents returned (commonly used in pagination).  
  _Example:_
  ```js
  db.users.find().limit(10);
  ```

## **3. MongoDB One-to-One Relationship**

- Example: A **User** has **one Profile**.

  ```js
  const userSchema = new mongoose.Schema({
    username: String,
    email: String,
    profileID: {
      type: mongoose.Schema.Types.ObjectId,
      ref: "UserProfile", // Reference schema name
    },
  });
  ```

- To retrieve the related profile data, use:
  ```js
  User.find().populate("profileID");
  ```

## **4. MongoDB One-to-Many Relationship**

- Example: **Author → Book** (One author can have many books, but each book belongs to one author).

  ```js
  const authorSchema = new mongoose.Schema({
    name: String,
    books: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Book",
      },
    ],
  });

  const bookSchema = new mongoose.Schema({
    title: String,
    author: {
      type: mongoose.Schema.Types.ObjectId,
      ref: "Author",
    },
  });
  ```

> 📌 Note:  
> Always create the **author first** before creating books, since each book must reference an existing author.

## **5. MongoDB Many-to-Many Relationship**

- Example: **Student ↔ Course**

  ```js
  const studentSchema = new mongoose.Schema({
    name: String,
    courses: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Course",
      },
    ],
  });

  const Student = mongoose.model("Student", studentSchema);

  const courseSchema = new mongoose.Schema({
    name: String,
    students: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Student",
      },
    ],
  });

  const Course = mongoose.model("Course", courseSchema);
  ```

  ```js
  db.users.find({ username: { $regex: "john", $options: "i" } });
  ```

## **7. Array Querying in MongoDB**

### 🔹 Find documents that have **both “electronics” and “home”**

```js
db.products.find({
  tags: { $all: ["electronics", "home"] },
});
```

✅ Matches any array that contains both `"electronics"` and `"home"`, regardless of order or extra elements.

---

### 🔹 Find documents that have **only “electronics”**

```js
db.products.find({
  tags: ["electronics"],
});
```

✅ Matches arrays that **only** contain `"electronics"`, and nothing else.

---

### 🔹 Difference Between Queries

| Query                                         | Matches                    | Description                |
| --------------------------------------------- | -------------------------- | -------------------------- |
| `{ tags: ["electronics", "home"] }`           | Exact order and match only | Must be exactly same array |
| `{ tags: { $all: ["electronics", "home"] } }` | Order doesn’t matter       | Must contain both          |
| `{ tags: { $in: ["electronics", "home"] } }`  | At least one matches       | Any one present            |

---

## **8. Advanced Array Matching**

### 🔹 Find documents where `tags` **only contain** `"electronics"` and `"home"` (no extra or missing)

```js
db.products.find({
  tags: { $all: ["electronics", "home"] },
  $expr: { $eq: [{ $size: "$tags" }, 2] },
});
```

✅ Matches arrays that contain **exactly two elements**, `"electronics"` and `"home"`, in any order.

### 🧾 Example:

```js
{ name: "TV", tags: ["electronics", "home"] } ✅
{ name: "Oven", tags: ["home", "electronics"] } ✅
{ name: "Laptop", tags: ["electronics"] } ❌
{ name: "Fridge", tags: ["home", "electronics", "kitchen"] } ❌
```
