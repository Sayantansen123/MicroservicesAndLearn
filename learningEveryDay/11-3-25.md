## **2. MongoDB Query Modifiers**

- **`skip()`** → Skips a specified number of documents before returning results.  
  _Example:_
  ```js
  db.users.find().skip(5);
  ```
- **`sort()`** → Sorts documents in ascending or descending order.  
  _Example:_
  ```js
  db.users.find().sort({ age: -1 }); // -1 = Descending, 1 = Ascending
  ```
- **`count()`** → Counts the number of matching documents.  
  _Example:_
  ```js
  db.users.find().count();
  ```
- **`limit()`** → Limits the number of documents returned (commonly used in pagination).  
  _Example:_
  ```js
  db.users.find().limit(10);
  ```

## **3. MongoDB One-to-One Relationship**

- Example: A **User** has **one Profile**.

  ```js
  const userSchema = new mongoose.Schema({
    username: String,
    email: String,
    profileID: {
      type: mongoose.Schema.Types.ObjectId,
      ref: "UserProfile", // Reference schema name
    },
  });
  ```

- To retrieve the related profile data, use:
  ```js
  User.find().populate("profileID");
  ```

## **4. MongoDB One-to-Many Relationship**

- Example: **Author → Book** (One author can have many books, but each book belongs to one author).

  ```js
  const authorSchema = new mongoose.Schema({
    name: String,
    books: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Book",
      },
    ],
  });

  const bookSchema = new mongoose.Schema({
    title: String,
    author: {
      type: mongoose.Schema.Types.ObjectId,
      ref: "Author",
    },
  });
  ```

> 📌 Note:  
> Always create the **author first** before creating books, since each book must reference an existing author.

## **5. MongoDB Many-to-Many Relationship**

- Example: **Student ↔ Course**

  ```js
  const studentSchema = new mongoose.Schema({
    name: String,
    courses: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Course",
      },
    ],
  });

  const Student = mongoose.model("Student", studentSchema);

  const courseSchema = new mongoose.Schema({
    name: String,
    students: [
      {
        type: mongoose.Schema.Types.ObjectId,
        ref: "Student",
      },
    ],
  });

  const Course = mongoose.model("Course", courseSchema);
  ```

## **7. Array Querying in MongoDB**

### 🔹 Find documents that have **both “electronics” and “home”**

```js
db.products.find({
  tags: { $all: ["electronics", "home"] },
});
```

✅ Matches any array that contains both `"electronics"` and `"home"`, regardless of order or extra elements.

---

### 🔹 Find documents that have **only “electronics”**

```js
db.products.find({
  tags: ["electronics"],
});
```

✅ Matches arrays that **only** contain `"electronics"`, and nothing else.

---

### 🔹 Difference Between Queries

| Query                                         | Matches                    | Description                |
| --------------------------------------------- | -------------------------- | -------------------------- |
| `{ tags: ["electronics", "home"] }`           | Exact order and match only | Must be exactly same array |
| `{ tags: { $all: ["electronics", "home"] } }` | Order doesn’t matter       | Must contain both          |
| `{ tags: { $in: ["electronics", "home"] } }`  | At least one matches       | Any one present            |

---

## **8. Advanced Array Matching**

### 🔹 Find documents where `tags` **only contain** `"electronics"` and `"home"` (no extra or missing)

```js
db.products.find({
  tags: { $all: ["electronics", "home"] },
  $expr: { $eq: [{ $size: "$tags" }, 2] },
});
```

✅ Matches arrays that contain **exactly two elements**, `"electronics"` and `"home"`, in any order.

### 🧾 Example:

```js
{ name: "TV", tags: ["electronics", "home"] } ✅
{ name: "Oven", tags: ["home", "electronics"] } ✅
{ name: "Laptop", tags: ["electronics"] } ❌
{ name: "Fridge", tags: ["home", "electronics", "kitchen"] } ❌
```

## 9. 🧠 MongoDB Indexing and `createIndexes()` in Mongoose

### 🔍 What Is Indexing?

Indexing in MongoDB is a technique to **speed up data retrieval**.  
Without an index, MongoDB performs a **collection scan**, checking every document to find the matching record.  
With an index, MongoDB uses a **B-tree data structure** to quickly locate data, similar to how an index in a book helps you find a page faster.

---

### ⚙️ How Indexing Works

When an index is created on a field (like `email`), MongoDB stores sorted keys and their references to actual documents.  
This allows the database to locate matching entries in **O(log n)** time instead of **O(n)**.

**Example:**

```js
db.users.find({ email: "user@example.com" });
```

- Without index → MongoDB scans every document.
- With index → MongoDB looks in the B-tree, finds the matching key, and retrieves the document instantly.

---

### 🧩 Creating Indexes in Mongoose

**1️⃣ Define in Schema**

```js
const userSchema = new mongoose.Schema({
  email: { type: String, unique: true }, // creates a unique index
  username: { type: String, index: true }, // adds an index
});
```

When the model is compiled, Mongoose automatically runs:

```js
Model.createIndexes();
```

to apply the defined indexes in MongoDB.

**2️⃣ Manually Create an Index**

```js
User.createIndexes({ email: 1 }); // ascending order
```

Or in Mongo shell:

```js
db.users.createIndex({ email: 1 });
```

---

### 📊 Common Index Types

| Type               | Description              | Example                      |
| ------------------ | ------------------------ | ---------------------------- |
| **Single Field**   | Index on one field       | `{ email: 1 }`               |
| **Compound Index** | Index on multiple fields | `{ category: 1, price: -1 }` |
| **Text Index**     | For text search          | `{ description: "text" }`    |
| **Hashed Index**   | For sharded data         | `{ userId: "hashed" }`       |
| **Unique Index**   | Prevents duplicates      | `{ email: 1, unique: true }` |

---

### 🧪 Example: Performance

Without index:

```js
db.products.find({ category: "electronics" });
```

→ Scans all documents (slow)

With index:

```js
db.products.createIndex({ category: 1 });
```

→ Uses B-tree to jump directly to matches (fast)

---

### ⚠️ Downsides of Too Many Indexes

- Increases **memory and storage** usage.
- Slows down **insert/update/delete** because indexes must be updated too.
- Use indexes only on fields queried frequently.
